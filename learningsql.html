Learning SQL

Home

SQL (Structured Query Language) is the standard language used to communicate with databases. It is used to manage and manipulate relational databases by querying, inserting, updating, and deleting data. SQL is an essential skill for anyone working with databases or involved in data analysis, data science, or backend development.

Basics

  • SELECT Statement - Retrieves data from a database. SELECT column1, column2 FROM table_name;
  • WHERE Clause - Filters results based on conditions. SELECT * FROM table_name WHERE condition;
  • INSERT INTO - Adds new data to a table. INSERT INTO table_name (column1, column2) VALUES (value1, value2);
  • UPDATE Statement - Modifies existing data in a table. UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition;
  • DELETE Statement - Removes data from a table. DELETE FROM table_name WHERE condition;
  • ORDER BY - Sorts query results. SELECT * FROM table_name ORDER BY column_name [ASC|DESC];
  • LIMIT - Restricts the number of rows returned. SELECT * FROM table_name LIMIT 10;
  • AND/OR - Combines multiple conditions. SELECT * FROM table_name WHERE condition1 AND condition2;
  • LIKE - Searches for a specified pattern in a column. SELECT * FROM table_name WHERE column_name LIKE 'pattern';
  • BETWEEN - Filters data within a range. SELECT * FROM table_name WHERE column_name BETWEEN value1 AND value2;

Intermediate

  • JOIN - Combines rows from two or more tables based on a related column. SELECT columns FROM table1 JOIN table2 ON table1.column = table2.column;
  • INNER JOIN - Returns records that have matching values in both tables. SELECT columns FROM table1 INNER JOIN table2 ON table1.column = table2.column;
  • LEFT JOIN - Returns all records from the left table and matching records from the right table. SELECT columns FROM table1 LEFT JOIN table2 ON table1.column = table2.column;
  • GROUP BY - Groups rows that have the same values into summary rows. SELECT COUNT(*), column FROM table_name GROUP BY column;
  • HAVING - Filters groups of rows after a GROUP BY. SELECT COUNT(*), column FROM table_name GROUP BY column HAVING COUNT(*) > 1;
  • DISTINCT - Returns only unique values. SELECT DISTINCT column FROM table_name;
  • ALIAS - Renames a column or table for easier reference. SELECT column AS alias_name FROM table_name;
  • IN - Allows you to specify multiple values in a WHERE clause. SELECT * FROM table_name WHERE column IN (value1, value2, value3);
  • COUNT, SUM, AVG, MAX, MIN - Aggregate functions. SELECT COUNT(*), AVG(column) FROM table_name;
  • CASE WHEN - Provides conditional logic in queries. SELECT column, CASE WHEN condition THEN result END FROM table_name;

Advanced

  • Subqueries - A query inside another query. SELECT column FROM table_name WHERE column IN (SELECT column FROM table_name WHERE condition);
  • UNION - Combines the result of two or more SELECT statements. SELECT column FROM table1 UNION SELECT column FROM table2;
  • INDEXES - Improves the speed of data retrieval operations. CREATE INDEX index_name ON table_name (column);
  • TRANSACTIONS - Manages a set of SQL queries as a single unit. BEGIN TRANSACTION; -- Starts a transaction COMMIT; -- Saves the changes ROLLBACK; -- Reverts the changes
  • Stored Procedures - Precompiled SQL code that can be executed. CREATE PROCEDURE procedure_name AS SELECT * FROM table_name;
  • Triggers - Automatically executes a query when certain events occur. CREATE TRIGGER trigger_name BEFORE INSERT ON table_name FOR EACH ROW BEGIN ... END;
  • Views - Virtual tables based on the result of a query. CREATE VIEW view_name AS SELECT column FROM table_name WHERE condition;
  • Locking - Prevents multiple transactions from interfering with each other. SELECT column FROM table_name FOR UPDATE;
  • Normalization - Organizing data to reduce redundancy and improve integrity. -- Breaking a single table into smaller tables with relationships
  • Foreign Keys - Ensures referential integrity between tables. CREATE TABLE table_name (column INT, FOREIGN KEY (column) REFERENCES other_table(column));

Expert

  • Partitioning - Divides a table into smaller, more manageable pieces. CREATE TABLE table_name PARTITION BY RANGE (column);
  • Sharding - Distributes data across multiple databases. -- Custom sharding logic based on application
  • Database Replication - Copies data across multiple servers. -- Configuring master-slave replication in MySQL
  • Materialized Views - Precomputed views that store data. CREATE MATERIALIZED VIEW view_name AS SELECT column FROM table_name;
  • Query Optimization - Enhances the performance of SQL queries. EXPLAIN SELECT * FROM table_name WHERE condition;
  • Full-text Search - Search for text data in columns. SELECT * FROM table_name WHERE MATCH(column) AGAINST ('search_term');
  • Database Clustering - Grouping multiple servers to increase database availability. -- Configuring database cluster
  • Stored Functions - Similar to stored procedures but returns a value. CREATE FUNCTION function_name() RETURNS INT AS BEGIN RETURN 1; END;
  • Query Caching - Storing query results to speed up repeated queries. -- Enable caching in the database server configuration

Master

  • Data Warehousing - Storing and analyzing large volumes of historical data. -- Data warehousing concepts such as ETL (Extract, Transform, Load)
  • Database Tuning - Optimizing databases for performance. -- Configuring server settings and index optimization
  • Big Data Integration - Managing and analyzing large datasets with SQL. -- Using SQL to query big data tools like Hadoop and Spark
  • Multi-Tenant Databases - Supporting multiple customers or users in a single database. -- Using partitioning or schemas to separate data
  • Advanced Security - Implementing encryption, user authentication, and access controls. -- Using SSL, role-based access, and encryption for data security
  • Backup and Recovery - Ensuring data durability and availability. -- Configuring automatic backups and point-in-time recovery
  • Event-Driven Architecture - Building reactive systems using SQL. -- SQL integration with event processing systems
  • Cloud Databases - Implementing SQL in cloud environments (AWS, Azure, Google Cloud). -- Configuring and scaling cloud SQL instances

Compilers and Tools for SQL

  • MySQL - Open-source relational database management system (RDBMS).
  • PostgreSQL - Advanced open-source RDBMS known for its scalability.
  • SQL Server - Microsoft's RDBMS offering with strong support for business intelligence.
  • Oracle Database - A widely used RDBMS for enterprise-level applications.
  • SQLite - Lightweight, serverless SQL database engine.
  • MariaDB - A community-driven fork of MySQL.
  • DB2 - IBM's relational database system.
  • Amazon RDS - Managed relational database service in AWS.
  • Google Cloud SQL - Fully managed SQL databases in Google Cloud.
  • DBVisualizer - SQL database management and query tool.
